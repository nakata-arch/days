# ② ミッション登録ページの新規実装

```
DAYS-projectにフランクリンプランナーの「ミッション登録」機能を追加してください。

【プロジェクト構成】
- Next.js + TypeScript（App Router）
- Firebase App Hosting + Firestore
- src/app/ にページ、src/components/ にコンポーネント
- src/firebase/ にFirebase操作、src/hooks/ にカスタムフック

【作りたいもの】
初回ログイン後に表示される「初期設定ページ」（/setup）を作成してください。
ここでユーザーが自分のミッション・役割・目標を登録します。

【登録する内容（3層構造）】
1. 役割（Role）
   - 例：「父親として」「管理職として」「学習者として」
   - 複数登録可能（最大5つ）
   - テキスト入力

2. 長期目標（Goal）
   - 各役割に紐づく目標
   - 例：「家族との時間を週10時間確保する」
   - 役割ごとに1つ

3. 今週のフォーカス（WeeklyFocus）
   - 今週特に意識したいこと
   - 自由記述（1〜3行）
   - 毎週更新可能

【Firestoreデータ構造】
users/{userId}/mission
  - roles: [{ id, name, goal }]
  - weeklyFocus: string
  - updatedAt: timestamp

【UX要件】
- シンプルで迷わないフォーム
- 入力項目は最小限（考えさせない）
- 登録後はホーム画面へ遷移
- 後から編集できる（設定ページへのリンク）
- スマホで使いやすいレイアウト

【デザイン方針】
- 既存のTailwindクラスと統一感を持たせる
- 落ち着いたビジネス向けのトーン
- 入力フォームは大きめのタップターゲット

【進め方のルール】
- まず現在のapp/のページ構成を確認してください
- 実装計画を日本語で出してOKをもらってから実装してください
- 既存のコンポーネントを再利用してください
- TypeScriptの型定義も追加してください
```
