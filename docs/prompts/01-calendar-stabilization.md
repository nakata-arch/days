# ① カレンダー連携の安定化

```
現在のコードを読んで、Googleカレンダー連携（OAuth）の安定化を行ってください。

【プロジェクト構成】
- Next.js + TypeScript（App Router）
- Firebase App Hosting
- src/firebase/ にFirebase関連コード
- src/hooks/ にカスタムフック

【確認・修正してほしい点】
まずコードを読んで現状を把握し、以下の順番で問題を確認・修正してください。

1. OAuthスコープの確認
   - Google Calendar API に必要なスコープが含まれているか
   - 必要スコープ: https://www.googleapis.com/auth/calendar.readonly

2. トークンリフレッシュ処理
   - アクセストークン期限切れ時に自動リフレッシュされるか
   - リフレッシュトークンが正しく保存・利用されているか
   - エラー時に再認証を促す処理があるか

3. エラーハンドリング
   - カレンダーAPI呼び出し失敗時のエラー処理
   - ネットワークエラー・レート制限への対応
   - ユーザーへのわかりやすいエラーメッセージ表示

4. イベント取得範囲
   - 当日を含む過去の予定が正しく取得されているか
   - タイムゾーン（Asia/Tokyo）が正しく処理されているか

【進め方のルール】
- まず現状のコードを読んで問題点を日本語で報告してください
- 修正計画を出してOKをもらってから実装してください
- 1ステップずつ進めてください
- 既存の動いている機能は壊さないでください
```
